# Free QR Code Generator

A free, ad-free QR code generator that turns a URL into a scannable QR code
**entirely in your browser**. No backend, no tracking — the URLs you encode
never leave your device.

**Live:** https://currylaksa.github.io/qrcode_generator_free/

## Features

- Paste a URL → get a scannable QR code
- **Lenient input:** bare domains like `example.com` are normalised to
  `https://example.com`; clearly-invalid input is rejected with a helpful message
- **Download** as PNG (universal) or SVG (vector, scales for print)
- **Size control:** Small (256px) / Medium (512px) / Large (1024px)
- A quiet "Support this free tool" tip-jar footer

## How it works

The whole app is static files served by GitHub Pages — there is no server
(see [ADR-0001](docs/adr/0001-client-side-only.md)). QR generation runs in the
browser via a vendored copy of the
[`qrcode-generator`](https://www.npmjs.com/package/qrcode-generator) library
(`vendor/qrcode.js`, no CDN). PNG export is painted from the QR module grid onto
a canvas, because the library's own data URL is a GIF, not a PNG
(see [ADR-0002](docs/adr/0002-png-via-canvas.md)).

There is **no build step** — the files you see are the files that ship.

## Project layout

```
index.html          The page: input, size control, preview, downloads, footer
vendor/qrcode.js    Vendored QR library (exposes window.qrcode)
src/
  qr.js             generateQrSvg(qrLib, url) — URL → SVG string
  normalize.js      normalizeUrl(raw) — trim, prepend https://, validate
  download.js       qrFilename() and qrMatrix() (the PNG/SVG source of truth)
  size.js           sizedSvg(svg, px) — resize via the SVG root width/height
  app.js            DOM glue wiring it all together
test/               Vitest tests (run in a happy-dom environment)
docs/adr/           Architecture decision records
CONTEXT.md          Domain glossary (QR Code vs Support QR, etc.)
bankQR.JPG          The Support QR shown in the footer (see below)
```

## Development

Tests use [Vitest](https://vitest.dev/) with a DOM environment. These are
**dev-only** dependencies; nothing is bundled into the deployed site.

```bash
npm install      # install dev dependencies
npm test         # run the test suite once
npm run test:watch
```

To preview locally, serve the folder over HTTP (opening `index.html` via
`file://` breaks ES-module loading):

```bash
python3 -m http.server 8000
# then open http://localhost:8000/
```

## Deploying

GitHub Pages serves from the `main` branch root (project page). Pushing to
`main` triggers a rebuild automatically. Asset paths are **relative** (`./…`)
so the site works from the `/qrcode_generator_free/` subpath.

## Swapping the Support QR

The footer shows `bankQR.JPG` at the repo root — a static image, never
generated by the app. To use your own, replace `./bankQR.JPG` and update the
`src`/`alt` of the `<img>` inside `index.html`'s `<footer id="support">`.

> Note: this image is committed to a public repo and shown on a public site,
> so whatever it contains (e.g. an account name) is publicly visible.
